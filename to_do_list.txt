✅ VERIFIED FIXED — kept for record, no action needed
- Predictable RNG: shared.py now defines secure_randint/secure_choice/secure_random (secrets-based), used throughout games_easy/medium/hard/heavy/poker.py and shared.py item selection. No plain random.random()/randint()/choice() left in game-outcome code paths.
- Race conditions in stateful sessions (Mines, Tower, Shotgun, Ladder, Hi-Lo, Mystery Box, Russian Roulette, Blackjack): per-user asyncio.Lock dicts now guard session access (_mine_locks, _tower_locks, _shotgun_locks, _ladder_locks, _hilo_locks, _mystery_locks, _rr_locks, _bj_locks).
- Missing minimum raise validation in Poker: routes/games_poker.py enforces min_raise = max(big_blind, current_bet*2) and rejects anything below it.
- Memory leak in game sessions (no TTL): server.py has _cleanup_game_sessions() with SESSION_TTL_SECONDS=300, started in the app lifespan, sweeping abandoned sessions and refunding bets.
- Jackpot state inconsistency (in-memory vs DB): jackpot pot/entries are now persisted transactionally in a jackpot_entries table (main.py), not just a global in-memory dict.
- Trade-up race condition (double-spend on traded items): trade-up/quick-trade paths now use SELECT ... FOR UPDATE / FOR UPDATE SKIP LOCKED before deleting traded items (server.py, main.py).

- Rate limiting on payment/auth endpoints (from the 2026-07-02 security audit, not originally in this file): fixed.
